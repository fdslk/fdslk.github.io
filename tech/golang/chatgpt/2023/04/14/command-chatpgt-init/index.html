chatgpt 命令行工具 commandChat

github

知乎

简书

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;最近花了一些时间写了一个小项目，直接在命令行中调用openai platform提供的api。有人可能会问了，为啥chatgpt提供了网页版你不用，非的自己弄一个麻不麻烦？大家就接着往下看吧~

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;我觉得把这个是看个人需求的，如果有的人只是想玩一玩的话，web版的确实够用了。但是大家肯定还是其他的需求的吧，比如说我想把每次的聊天记录保存下来、比如说我想选择特定的模型聊天、比如说我想给出特定的prompt数据、再比如我想使用openai提供的一些很fancy的功能（画图，语音）。当然前两个肯定是可以实现的，因为在platform上提供了try it，直接调用api并且定义自己的数据。但是我是乐意把数据掌握在自己的手上。我想应该有一些小伙伴也想要把数据掌握在自己的手上吧，除此之外web版的GPT很容易无响应，需要重新刷新页面，使用api调用的方式会更加的稳定。

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;下面我给大家简单说说我的这个工具吧，因为是想要在命令行中使用，然后又结合了chatgpt的聊天功能，所以取名为commandChat。工具主要基于Golang的cobra开发，cobra是一个cmd交互的go package，被很多的Go生态应用所使用。在cobra的github站点 上列出了很多的使用项目，其中比较有名的就是Kubernetes。基于cobra可以方便地扩展自己的交互命令，只需要在本地安装cobra-cli就可以非常方便的集成新的命令。具体的使用方式，大家可以可参考cobra的官方文档 。看到这里，大家可能又会有疑问了，openai官方不是提供了python和nodejs的package，你又为啥要用golang重写一些底层的轮子呢。我一句话应该就可以说服你，Go的跨平台打包非常方便，只需要在当前平台上下载代码，然后安装Go环境，在需要打包的项目下执行go build，一个可执行文件就生成了。还有一个就是Go对多线程，高并发的一些优势，比较适合我后续feature的开发。

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;commandChat现在有哪些功能呢，第一个是核心功能，调用openai提供的LLM模型和gpt对话。方式也很简单，大家如果在本地没有安装Go的环境可以下载 我提供的mac环境可执行文件。暂时在mac上开发，所以打出来的包是mac的executor file，如果大家有更多的需要我也可以在windows打包。为了是大家使用的更加方便，可以把commandChat放到环境目录下。如果在windows下大家可以参考《How to Add Executables to your PATH in Windows》 ，如果在mac下可能更加方便，执行sudo cp commandChat /usr/local/bin/， 就可以方便的玩耍了。commandChat默认的聊天模型是gpt-3.5-turbo，每次会将最新的两条聊天记录作为当前聊天的context。当当前聊天结束的时候，会将本次聊天的记录保存到和commandChat在相同目录下的historys文件夹下，如果没有给聊天记录命名，将会以当前聊天的第一个问题作为我们的文件名称。

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;第二个功能是修改commandChat的设置，运行命令commandChat setting, 进入命令之后会打印出当前的设置。现在的设置文件使用json格式来存储，暂时不接受其他的格式。通过编辑设置文件，可以切换聊天模型。如果说，你要切换到davinci模型的，输入

然后再输入save保存修改。这样切换到了davinci模型，使用davinci模型时，commandChat没有提供上下文，也就是说每次只是一次新的对话没有上下文。
